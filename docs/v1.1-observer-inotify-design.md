# V1.1 Observer inotify 设计文档

## 背景

当前 Observer 使用 60s polling 监控 state.json 变化，导致通知延迟最高 60s。
V1.1 #2 目标：升级为 inotify 文件监控，降低延迟至 <1s。

## 设计目标

1. **低延迟** — 文件变化后 <1s 触发通知
2. **跨平台** — Linux 用 inotify，macOS/Windows 用 polling fallback
3. **向后兼容** — 不破坏现有 Observer 接口
4. **可测试** — 单元测试覆盖所有路径

## 架构

```
FileWatcher Protocol
  ├── InotifyWatcher (Linux) — inotify + epoll
  ├── PollingWatcher (fallback) — 60s polling
  └── MockWatcher (testing) — 手动触发事件

Observer
  ├── FileWatcher watcher
  ├── check_liveness(state)
  └── send_notification(event)
```

## 接口设计

```python
from typing import Protocol, Literal
from pathlib import Path
from dataclasses import dataclass

@dataclass(frozen=True)
class FileEvent:
    """文件变化事件。"""
    path: Path
    event_type: Literal["created", "modified", "deleted", "overflow"]
    timestamp: str  # ISO 8601

class FileWatcher(Protocol):
    """文件监控器接口。"""
    
    def watch(self, paths: list[Path]) -> None:
        """开始监控指定文件/目录列表。"""
        ...
    
    def next_event(self, timeout_seconds: float = 1.0) -> FileEvent | None:
        """阻塞等待下一个事件，超时返回 None。
        
        返回 event_type="overflow" 表示队列满，调用方应全量重新扫描。
        """
        ...
    
    def stop(self) -> None:
        """停止监控。"""
        ...
```

## InotifyWatcher 实现要点

1. **使用 inotify 系统调用** — `inotify_init1(IN_NONBLOCK | IN_CLOEXEC)`
2. **监控父目录而非文件** — 观测 `.unison/` 目录，按文件名过滤事件（避免 atomic_write 的 inode 变更陷阱）
3. **监控事件类型** — `IN_CLOSE_WRITE | IN_CREATE | IN_DELETE | IN_MOVED_TO | IN_MOVED_FROM`
4. **epoll 等待** — `epoll_create1() + epoll_ctl(ADD) + epoll_wait()`
5. **信号处理** — `signal.signal(signal.SIGINT, handler)` 设置退出标志位，事件循环检查标志
6. **队列满处理** — 检测到 `IN_Q_OVERFLOW` 时返回特殊事件 `FileEvent(path=Path("."), event_type="overflow", timestamp=...)`，Observer 全量重新扫描
7. **观测数限制** — 捕获 `OSError(errno=ENOSPC)` 并降级到 PollingWatcher，记录警告

## PollingWatcher 实现要点

1. **记录文件 mtime** — `path.stat().st_mtime`
2. **每 5s 扫描**（非 Linux 平台）— 比较 mtime 变化（<1s 延迟仅 Linux 规格，macOS/Windows 为 5s）
3. **检测新文件** — 对比已知文件列表
4. **检测删除** — `path.exists()` 返回 False
5. **无外部依赖** — 纯标准库实现，`watchdog` 作为 v1.2 增强项考虑

## Observer 集成

```python
class Observer:
    def __init__(
        self, 
        world: World, 
        stall_threshold_seconds: int = 300,
        watcher: FileWatcher | None = None
    ):
        self.world = world
        self.stall_threshold_seconds = stall_threshold_seconds
        self.watcher = watcher or self._create_default_watcher()
    
    def _create_default_watcher(self) -> FileWatcher:
        """根据平台选择 watcher。"""
        import sys
        if sys.platform == "linux":
            try:
                return InotifyWatcher()
            except OSError as e:
                if e.errno == errno.ENOSPC:
                    logger.warning("inotify watches exhausted, falling back to polling")
                    return PollingWatcher(interval_seconds=5)
                raise
        else:
            return PollingWatcher(interval_seconds=5)
    
    def run(self) -> None:
        """阻塞循环。监控 state.json + notifications.jsonl。Ctrl-C 退出。"""
        # 监控父目录，按文件名过滤（避免 atomic_write inode 陷阱）
        paths = [self.world.unison_dir]
        self.watcher.watch(paths)
        
        self._running = True
        while self._running:
            try:
                event = self.watcher.next_event(timeout_seconds=1.0)
            except StopIteration:
                # watcher 被停止
                break
            
            if event is None:
                continue
            
            # 处理 overflow 事件（inotify 队列满）
            if event.event_type == "overflow":
                # 全量重新扫描 state + notifications
                self._full_rescan()
                continue
            
            # 过滤非目标文件事件
            if event.path.name not in ("state.json", "notifications.jsonl"):
                continue
            
            # 检测 phase transition
            if event.path.name == "state.json":
                if not self.world.state_file.exists():
                    # state.json 缺失 → 关键路径丢失，退出
                    raise RuntimeError("state.json missing, cannot continue")
                
                state = State.load(self.world.state_file)
                if not self.check_liveness(state):
                    self._write_notification(Notification(
                        timestamp=datetime.now(timezone.utc).isoformat(),
                        phase=state.phase,
                        severity="warn",
                        title="Session stalled",
                        body=f"No activity for 5min+ in phase {state.phase}"
                    ))
            
            # 检测新通知
            if event.path.name == "notifications.jsonl":
                if not self.world.notifications_file.exists():
                    # notifications.jsonl 缺失 → 非关键，重建即可
                    self.world.notifications_file.touch()
                else:
                    # 读取新增通知 → Discord
                    self._process_new_notifications()
    
    def stop(self) -> None:
        """停止 Observer 循环。"""
        self._running = False
        self.watcher.stop()
```

## MockWatcher 接口

```python
class MockWatcher:
    """测试用 watcher，手动注入事件。"""
    
    def __init__(self):
        self._events: list[FileEvent] = []
        self._watched_paths: list[Path] = []
        self._stopped = False
    
    def watch(self, paths: list[Path]) -> None:
        self._watched_paths = paths
    
    def inject_event(self, event: FileEvent) -> None:
        """测试注入事件。"""
        self._events.append(event)
    
    def next_event(self, timeout_seconds: float = 1.0) -> FileEvent | None:
        if self._stopped:
            return None
        if self._events:
            return self._events.pop(0)
        time.sleep(min(timeout_seconds, 0.1))
        return None
    
    def stop(self) -> None:
        self._stopped = True
```

## 依赖

- Linux: `ctypes` 调用 inotify/epoll 系统调用（标准库）
- macOS: `watchdog` 库（可选，fallback 到 polling）
- 无新外部依赖（V1 约束：纯标准库 + pyyaml）

## 风险

1. **inotify 队列满** — 大量文件变化时可能丢失事件 → 检测到 `IN_Q_OVERFLOW` 时返回 overflow 事件，Observer 全量重新扫描
2. **信号中断** — Ctrl-C 时 epoll_wait 返回 EINTR → Python 3.5+ (PEP 475) 自动重试，通过 `_running` 标志位检查退出
3. **文件被删除** — state.json 缺失视为致命错误（raise RuntimeError），notifications.jsonl 缺失自动重建（touch）
4. **inotify 观测数限制** — 捕获 `ENOSPC` 并降级到 PollingWatcher
